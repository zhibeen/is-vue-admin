## 🎓 文档维护规范

### 以后创建文档时

1. **确定文档类型**
   - 使用指南 → `docs/guides/`
   - 架构设计 → `docs/architecture/`
   - 功能模块 → `docs/features/{module}/`
   - API 文档 → `docs/api/`
   - 第三方集成 → `docs/integrations/`

2. **使用标准文件名**
   ```
   # ✅ 好的命名
   deployment-guide.md
   database-schema.md
   api-authentication.md
   
   # ❌ 差的命名
   部署指南.md
   物流对账与财务应付单系统_开发完成报告.md
   ```

3. **临时文档处理**
   - 实施指南完成后，提取有价值内容到正式文档
   - 删除或归档纯临时文档
   - 不要在项目根目录或 `docs/` 中保留临时文档

4. **更新文档索引**
   - 新增文档后，更新 `docs/README.md`
   - 更新模块的 README

## 🆘 问题排查

### 问题 1: 脚本执行失败

**原因**: 可能不在项目根目录

**解决**:
```bash
# 确认当前目录
pwd  # Linux/Mac
cd   # Windows

# 应该看到 README.md、backend、frontend 目录
```

### 问题 2: 文件已存在

**原因**: 之前可能已经运行过脚本

**解决**:
- 检查 `docs/` 目录是否已创建
- 检查 `archive/temp/` 是否已有文件
- 根据需要手动整理

### 问题 3: 文档链接失效

**原因**: 迁移后文档路径变化

**解决**:
- 搜索项目中的文档链接
- 更新为新的路径
- 使用相对路径而不是绝对路径
